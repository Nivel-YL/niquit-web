# Достоверность контента — архитектура исследования

> Этот документ заменяет раздел "Исследование и написание" в AI_EDITOR_AUTOMATION_SPEC.md целиком.

---

## Проблема, которую мы решаем

До этой версии: функция `research()` вызывалась 5 раз независимо — по одному на каждый язык.
Это создавало два риска:

1. Пять независимых поисков могли найти разные цифры для одного и того же явления (например, «224 тысячи» в одной языковой версии и «300 тысяч» в другой).
2. Перерасход на поиск: 5 × поиск = до 10 поисковых вызовов на тему.

---

## Новая архитектура (действующая)

### Шаг 1 — Одно общее исследование на тему

Один поисковый вызов (не пять). Собирает универсальные факты: статистику, механизмы, актуальные данные, каждый факт с источником и датой.

Вызов модели: `claude-haiku-4-5`, `max_uses: 3`.

В конце ответа модель должна написать:
```
NEEDS_LOCAL_RESEARCH: ru,de   (или "none")
```

### Шаг 2 — Локальное исследование (только если нужно)

Отдельный точечный поиск только для языков, где он нужен: юридический статус, локальная статистика, культурный контекст (например, DiGA в Германии).

Вызов модели: `claude-haiku-4-5`, `max_uses: 2`. По одному вызову на язык, только для тех, что перечислены в `NEEDS_LOCAL_RESEARCH`.

### Шаг 3 — 5 параллельных вызовов на написание

Каждый из 5 вызовов получает одинаковую базу фактов + локальное дополнение своего языка (если есть).

Вызов модели: `claude-sonnet-5`, `max_tokens: 4096`.

### Шаг 4 — 5 параллельных фактических аудитов

После написания каждой статьи — дешёвый вызов Haiku без поиска. Извлекает каждое конкретное число/факт/правовое утверждение из написанного текста и сверяет с тем, что было в исследовательских фактах.

Результат: таблица VERIFIED / DISCREPANCY / UNVERIFIED — сохраняется рядом с черновиком в `docs/fact-audits/{slug}/{lang}.md`.

---

## Обязательные правила достоверности

### Числа и статистика

- Каждое конкретное число должно быть подтверждено **минимум двумя независимыми источниками**.
- Если источники расходятся — пишем диапазон («от X до Y»), не одну цифру.
- Каждый факт передаётся в этап написания вместе с источником и датой публикации.

### Свежесть источников

- Предпочтение: источники моложе 3 лет, особенно для правовых и регуляторных фактов.
- Если найденный источник старше 3 лет — явно помечаем: «актуально на [дата источника], могло измениться».

### Иерархия источников

1. Официальные органы здравоохранения и правительства (ВОЗ, Минздрав, CDC и аналоги)
2. Рецензируемые исследования (peer-reviewed)
3. Авторитетные новостные издания
4. Всё остальное — только если ничего лучше не найдено, с явной пометкой

### Правовые факты — отдельная категория

Любое утверждение о законности/запрете чего-либо должно содержать: «по состоянию на [дата источника]».
Не как неизменная истина. Правовые нормы меняются.

### Паритет между языковыми версиями — безопасность

**Правило:** Если в любой языковой версии статьи появляется информация, критически важная для безопасности читателя (законодательные запреты, требования к возрасту, медицинские предупреждения), — та же информация **обязательно** должна присутствовать во **всех** остальных языковых версиях, как минимум в форме общего предупреждения: «правовой статус зависит от страны».

**Примеры safety-critical фактов:**
- Продукт запрещён в конкретной стране (например, никотиновые паучи во Франции с апреля 2026 года)
- Ограничения по возрасту
- Предупреждения органов здравоохранения о конкретной опасности

**Ответственность редактора:** При публикации статьи с региональной safety-critical информацией — убедиться, что все языковые версии содержат эквивалентное предупреждение. Одноязычный safety-gap (есть в FR, нет в EN/RU/DE/ES) недопустим для опубликованных материалов.

**Пер-языковые данные для конкретных рынков** (статистика по отдельной стране, маркетинговые данные, местные бренды) — не относятся к safety-critical и могут различаться между языками.

---

## Правило «не является медицинским советом»

Применяется ко **всем** темам без исключения (не только к кластеру мифов).

Текст статьи может описывать механизмы, статистику, исследования. Но не должен формулировать «поэтому тебе стоит / не стоит делать X». Советы — это работа врача, не блога.

---

## Внутренние ссылки (реально работающая версия)

Этап написания получает на вход список **уже опубликованных** (не просто сгенерированных) статей: заголовок и слаг. Без этого инструкция «добавляй внутренние ссылки на предыдущие статьи» невыполнима.

Список обновляется автоматически при каждой публикации.

---

## Дедупликация внутри кластера

Если в кластере несколько статей подряд (например, 4 статьи о никотиновых паучах) — этап написания явно предупреждается: не повторяй одни и те же вводные объяснения, которые уже были в других статьях того же кластера. Ссылайся на них, не пересказывай.

---

## Фактический аудит — для ревью редактором

Короткий отдельный вызов (без поиска, Haiku) после написания каждой статьи:

- Извлекает из готового текста каждое число, статистику, правовой факт
- Сравнивает с тем, что было предоставлено из исследовательского этапа
- Если что-то в финальном тексте не совпадает с найденными фактами — помечает явно
- Результат: таблица в markdown

```
| Claim in article | Status | Source / Note |
|---|---|---|
| "около 300 тысяч жизней" | VERIFIED | Минздрав, 2023 |
| "каждый третий пользователь" | UNVERIFIED | Не найдено в исследовании |
```

Редактор проверяет не весь текст в поисках ошибки, а именно этот компактный список — отклонение видно сразу.

---

## Критерии выполнения

- [ ] Исследование темы выполняется один раз, не пять
- [ ] Каждое число в готовых статьях имеет источник и дату (видно в отдельном отчёте)
- [ ] Правовые факты помечены датой актуальности
- [ ] Ссылки между статьями кластера реально появляются в тексте
- [ ] Фактический отчёт создаётся автоматически для каждого черновика до того, как редактор его читает
